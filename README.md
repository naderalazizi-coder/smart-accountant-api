# Smart Accountant ERP — API | واجهة المحاسب الذكي الخلفية

خادم حقيقي (Node.js + Express + PostgreSQL) يسمح بتسجيل شركات متعددة، كل
شركة معزولة تماماً عن غيرها عبر Row-Level Security في قاعدة البيانات. تم
اختباره فعلياً: شركتان مسجَّلتان، كل واحدة أنشأت حساباتها الخاصة، وتأكّدنا
أن الشركة الثانية لا ترى بيانات الأولى إطلاقاً.

لا توجد بيانات وهمية — كل شركة تبدأ فارغة تماماً بعد التسجيل.

## التشغيل محلياً (على جهازك)

يحتاج فقط [Docker](https://www.docker.com/products/docker-desktop/) مثبّت:

```bash
docker compose up --build
```

بعد التشغيل:
- الخادم يعمل على: `http://localhost:3000`
- فحص الحالة: `curl http://localhost:3000/health`

## اختبار سريع عبر curl

```bash
# تسجيل شركة جديدة
curl -X POST http://localhost:3000/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "companyName": "My Company",
    "companyNameAr": "شركتي",
    "fullName": "Your Name",
    "email": "you@company.com",
    "password": "SecurePass123"
  }'
# الرد يحتوي على "token" — استخدمه في باقي الطلبات:
# Authorization: Bearer <token>

# تسجيل الدخول لاحقاً
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"you@company.com","password":"SecurePass123"}'
```

## نقاط النهاية (Endpoints) المتوفرة الآن

| الطريقة | المسار | الوصف |
|---|---|---|
| POST | `/auth/signup` | تسجيل شركة جديدة + أول مستخدم (مالك) |
| POST | `/auth/login` | تسجيل الدخول، يرجع JWT |
| GET | `/auth/me` | بيانات المستخدم الحالي |
| GET/POST | `/api/accounts` | دليل الحسابات (شجرة الحسابات) الخاص بالشركة |
| POST | `/api/periods/bootstrap` | إنشاء السنة المالية والفترات الشهرية (لازم قبل أي قيد) |
| GET/POST | `/api/journal-entries` | إنشاء قيود محاسبية متوازنة (مدين = دائن) |
| POST | `/api/journal-entries/:id/post` | ترحيل القيد |

كل هذه المسارات (ما عدا auth) تتطلب Header:
`Authorization: Bearer <token>`

## للحصول على رابط حقيقي يعمل على الإنترنت (خطوات كاملة من الجوال)

الخادم الآن يهيّئ قاعدة البيانات تلقائياً عند أول تشغيل (لا حاجة لتنفيذ
أوامر SQL يدوياً) — هذا يجعل النشر السحابي ممكناً بالكامل من متصفح
الجوال، بدون كمبيوتر أو أوامر Docker.

### الخطوات (باستخدام GitHub + Railway، مجاني للبداية)

1. **أنشئ حساب GitHub** (إذا ما عندك): على github.com من متصفح الجوال،
   بريد إلكتروني وكلمة مرور فقط.
2. **ارفع هذا المجلد كمستودع (repository)**:
   - من GitHub اضغط "+" ثم "New repository"، سمّه مثلاً `smart-accountant-api`.
   - داخل المستودع اضغط "Add file" → "Upload files"، واسحب/اختر كل ملفات
     هذا المجلد (بدون مجلد `node_modules` إذا موجود).
   - اضغط "Commit changes".
3. **أنشئ حساب Railway**: على railway.app، سجّل دخول باستخدام حساب GitHub
   نفسه (زر "Login with GitHub") — يربطهم ببعض تلقائياً.
4. **أنشئ مشروع جديد**: "New Project" → "Deploy from GitHub repo" →
   اختر المستودع اللي رفعته.
5. **أضف قاعدة بيانات**: داخل نفس المشروع اضغط "New" → "Database" →
   "Add PostgreSQL". Railway ينشئها تلقائياً ويعطيها متغير `DATABASE_URL`.
6. **اربط المتغير بخدمة الـ API**: افتح خدمة الـ API (المستودع اللي رفعته)
   → تبويب "Variables" → أضف متغير جديد باسم `DATABASE_URL` وقيمته:
   اضغط "Add Reference" واختر `DATABASE_URL` من قاعدة البيانات اللي أنشأتها.
   أضف أيضاً متغير `JWT_SECRET` بقيمة عشوائية طويلة (أي نص عشوائي لا أحد
   يعرفه).
7. **انتظر النشر** (Railway يبني وينشر تلقائياً بعد كل تعديل) — بعد
   انتهائه، من تبويب "Settings" → "Networking" اضغط "Generate Domain"،
   يعطيك رابط شبيه بـ `https://smart-accountant-api-production.up.railway.app`.
8. **افتح تطبيق الواجهة** (ملف `smart_accountant_app.html`)، وفي شاشة
   الدخول افتح "إعدادات متقدمة: عنوان الخادم" وضع نفس الرابط اللي حصلت
   عليه بدل `http://localhost:3000`.

من هذه اللحظة، أي شخص من أي جهاز (جوال أو كمبيوتر) يقدر يفتح ملف الواجهة
وينشئ حساب شركته الخاص، وبياناته محفوظة فعلياً على الإنترنت.

### بديل أبسط (إذا احتجت مساعدة مباشرة)

الخطوات فوق تقنية شوي حتى لو من الجوال. إذا تفضّل، أقدر أجهز لك نفس
الملفات بصيغة أسهل للرفع، أو تقدر تطلب من مطوّر يسويها لك خلال ٢٠ دقيقة
تقريباً — الكود جاهز بالكامل ومُختبر، الباقي خطوات نقر فقط.

## الخطوات التالية المنطقية

1. ربط الواجهة (لوحة التحكم) بهذا الخادم الفعلي بدل البيانات التجريبية.
2. نشر الخادم على استضافة حقيقية للحصول على رابط دائم.
3. إضافة وحدات: المبيعات، المشتريات، المخزون، الموارد البشرية (البنية
   جاهزة في `schema.sql`، تحتاج فقط مسارات API مثل التي بنيناها للمحاسبة).
4. إضافة نظام دعوة مستخدمين إضافيين لنفس الشركة (حالياً كل تسجيل جديد
   ينشئ شركة جديدة منفصلة).
